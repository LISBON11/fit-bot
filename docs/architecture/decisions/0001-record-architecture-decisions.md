# 1. Запись архитектурных решений (ADR)

Дата: 2026-02-22

## Статус

Принято

## Контекст

Проект FitBot со временем будет усложняться (бот -> API -> мобильное приложение). Нам необходимо зафиксировать историю архитектурных решений и причину выбора той или иной технологии (например, почему выбрали PostgreSQL, Prisma, grammY). Традиционный `docs/architecture/design/SYSTEM_DESIGN.md` подходит для высокоуровневой картины, но менее удобен для хронологической истории принятия решений.

## Решение

Мы будем использовать формат Architecture Decision Records (ADR).
Каждое значимое архитектурное решение (смена БД, разделение на микросервисы, изменение способа авторизации) будет описываться в отдельном файле в папке `docs/architecture/decisions/` с именем формата `NNNN-краткое-название.md`.

## Последствия

- Повысится прозрачность принятия решений для новых разработчиков (и AI-агентов).
- Придется тратить дополнительное время на написание ADR при каждом важном изменении.
- Решения из ADR со временем могут дополнять или изменять `docs/architecture/design/SYSTEM_DESIGN.md`.
