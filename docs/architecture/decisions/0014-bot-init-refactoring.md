# ADR-0014: Рефакторинг инициализации бота (bot.ts)

## Статус

Принято (Accepted)

## Контекст

Файл `src/bot/bot.ts` стал слишком объемным и содержал:

- Сложную bypass-логику отмены (Optimistic UI), смешанную с настройкой плагинов.
- Дублирование логики захвата блокировок (`lockUser`/`unlockUser`) для команд и сообщений.
- Неконсистентность между командой `/cancel` и кнопкой "Отмена".

## Решение

1. **Унификация отмены**: Создана утилита `src/bot/utils/cancelFlow.ts`, содержащая логику прерывания процесса (обновление UI, выход из conversation, удаление черновиков, очистка Redis и разблокировка).
2. **Безопасный вход**: Создан хелпер `enterWithLock` в `src/bot/utils/conversationHelpers.ts`, который инкапсулирует проверку блокировок и вход в диалог.
3. **Разделение ответственности**: Детали реализации вынесены в `handlers/callbackHandlers.ts` и `handlers/commandHandlers.ts`.

## Последствия

- В `bot.ts` теперь только чистая конфигурация.
- Логика отмены работает одинаково для команды `/cancel` и кнопки в UI.
- Исключена возможность двойного входа в диалоги через спам (блокировка в `enterWithLock`).
- Улучшена читаемость кода за счет использования паттерна Object Parameters и логирования pino.
