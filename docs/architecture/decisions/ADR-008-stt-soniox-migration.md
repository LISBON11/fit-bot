# ADR-008: Миграция STT с Deepgram Nova-3 на Soniox stt-async-v4

**Дата:** 2026-03-12  
**Статус:** Принято  
**Авторы:** AI-ассистент (Antigravity)

---

## Контекст

Бот использовал Deepgram Nova-3 для распознавания голосовых сообщений пользователей. Качество транскрипции русского языка оказалось неудовлетворительным — пользователи сообщают о значительном количестве ошибок, особенно на фитнес-терминологии.

Задача: выбрать и мигрировать на STT-провайдера с лучшим качеством для русского языка.

---

## Рассматриваемые варианты

| Провайдер                 | WER (RU)     | Цена/час | Сложность интеграции       |
| ------------------------- | ------------ | -------- | -------------------------- |
| **Soniox stt-async-v4**   | **~6.2%**    | $0.10    | Средняя (REST API)         |
| OpenAI gpt-4o-transcribe  | ~5-8%        | $0.36    | Низкая (уже есть SDK)      |
| Groq + Whisper Large-v3   | ~9.8%        | $0.111   | Низкая (OpenAI-compatible) |
| Google Chirp 3            | ~16.7%       | $0.96    | Средняя                    |
| Deepgram Nova-3 (текущий) | плохо для RU | $0.26    | —                          |

WER-данные из независимого бенчмарка на реальном YouTube-аудио (2025).

---

## Принятое решение

**Soniox stt-async-v4** — лучший WER для русского (6.2%), самый дешёвый из топа ($0.10/час). Модель v4 (январь 2026) заявляет human-parity accuracy для реального аудио.

**Ключевое преимущество:** поддержка `context.terms` — передаём список фитнес-терминов, что улучшает распознавание специализированной лексики (жим лёжа, становая тяга и т.д.).

---

## Реализация

### Паттерн транскрипции

Soniox работает асинхронно (не стриминг):

1. `POST /v1/files` — загрузка аудио-буфера → `id` (uuid)
2. `POST /v1/transcriptions` — создание задачи → `id`
   - **Важно:** Для модели `stt-async-v4` поле `file_id` должно быть обёрнуто в объект `payload`.
3. `GET /v1/transcriptions/{id}` — поллинг статуса (интервал 500ms, таймаут 30s)
4. `GET /v1/transcriptions/{id}/transcript` — получение текста из токенов
5. `DELETE /v1/transcriptions/{id}` + `DELETE /v1/files/{id}` — очистка

Для коротких голосовых Telegram (10-30 сек) процесс занимает ~1-3 секунды.

### Изменённые файлы

| Файл                                   | Изменение                                               |
| -------------------------------------- | ------------------------------------------------------- |
| `src/stt/soniox.stt.ts`                | **NEW** — реализация `SttService` через Soniox REST API |
| `src/stt/__tests__/soniox.stt.test.ts` | **NEW** — 6 unit-тестов                                 |
| `src/config/env.ts`                    | `DEEPGRAM_API_KEY` → `SONIOX_API_KEY`                   |
| `src/services/index.ts`                | DI: `DeepgramStt` → `SonioxStt`, тип → `SttService`     |
| `src/config/__tests__/env.test.ts`     | Обновлены тест-данные под новый ключ                    |
| `src/stt/deepgram.stt.ts`              | Сохранён для отката, отвязан от типизированного конфига |

### Фитнес-контекст

В `context.terms` передаются ~40 терминов: жим лёжа, становая тяга, приседания и пр.
Файл: `src/stt/soniox.stt.ts`, константа `FITNESS_TERMS`.

## Последствия

### Положительные

- Значительно лучшее качество распознавания русской речи
- Более низкая стоимость ($0.10 vs $0.26/час)
- Фитнес-контекст снижает ошибки на специализированных терминах
- Архитектура `SttService` позволяет легко откатиться или сменить провайдера

### Отрицательные / риски

- Soniox — относительно молодая компания (~2021), меньше community support
- Async-модель: ~1-3 сек дополнительной задержки vs мгновенный ответ Deepgram
- Требуется новый `SONIOX_API_KEY` в `.env`

### Откат

Для быстрого отката достаточно в `src/services/index.ts` заменить `SonioxStt` → `DeepgramStt` и восстановить `DEEPGRAM_API_KEY` в `env.ts`. Файл `deepgram.stt.ts` сохранён.
