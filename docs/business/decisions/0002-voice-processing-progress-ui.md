# 0002. Прогресс-статусы при обработке голосового сообщения

Дата: 2026-03-05

## Текущий статус

Принято

## Контекст

Обработка голосового сообщения — многоэтапный асинхронный процесс:
распознавание речи (STT, ~1-3 сек) → парсинг NLU (LLM, ~3-8 сек) →
создание упражнений в БД → сохранение тренировки → возможный disambiguation.

Без обратной связи пользователь воспринимает задержку как «бот завис»,
что ухудшает UX и снижает доверие к продукту.

## Рассматриваемые варианты

### Вариант A: `typing` индикатор (текущее решение)

Простой `chat_action = typing` без конкретики. Не информативен при задержках >3 сек.

### Вариант B: Статусное сообщение с шагами _(выбрано)_

Бот отправляет отдельное сообщение со списком шагов и обновляет его в реальном времени,
меняя эмодзи-статусы: ⬜ → ⏳ → ✅ / ⚫.

### Вариант C: Прогресс-бар текстовый

Единая строка вида `[████░░] 60%`. Менее информативен, чем список шагов.

## Решение

Реализован **Вариант B** — живое сообщение с 6 шагами:

| #   | Шаг                        | Эмодзи | Условие пропуска                                      |
| --- | -------------------------- | ------ | ----------------------------------------------------- |
| 1   | Речь в текст               | 🎙     | —                                                     |
| 2   | Понимаем тренировку        | 🧠     | —                                                     |
| 3   | Добавляем новые упражнения | 🏋     | ⚫ если все упражнения уже есть в базе                |
| 4   | Сохраняем в базу           | 💾     | —                                                     |
| 5   | Уточняем детали            | 🔍     | ⚫ если disambiguation не нужен                       |
| 6   | Публикуем в канал          | 📢     | Активируется только после подтверждения пользователем |

**Поведение при отмене:** статус-сообщение удаляется (без отдельного ⚫/❌ в трекере,
т.к. сообщение об отмене уже отображается в превью тренировки).

**Шаг 3 с подсписком:** если есть новые упражнения — под строкой шага появляется
список с индивидуальными статусами ⬜/⏳/✅ для каждого создаваемого упражнения.

**Реализация:** класс `ProgressTracker` (`src/bot/utils/progressTracker.ts`),
передаётся как опциональный параметр — не ломает совместимость с текстовым вводом
и режимом редактирования.

## Последствия

### Положительные

- Пользователь понимает, что происходит и не закрывает бот в ожидании
- При ошибке легче диагностировать на каком шаге всё встало
- Подсписок упражнений создаёт ощущение «живой» работы ИИ

### Отрицательные

- Дополнительные API-вызовы `editMessageText` (~6-10 шт.): незначительная нагрузка
- Трекер остаётся над превью — требует от пользователя скроллить вниз к кнопкам preview
